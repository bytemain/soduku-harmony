# 📋 数独标准格式与导入导出支持

> 调研业界通用的数独谜题交换格式，分析各格式的特点和适用场景，
> 为本项目实现标准化导入/导出功能提供技术方案。

---

## 一、格式全景

数独社区经过多年发展，形成了多种谜题交换格式。虽然没有统一的 "官方标准"，
但以下几种格式已被广泛接受，成为事实标准：

| 格式 | 类型 | 适用场景 | 通用程度 | 推荐支持 |
|------|------|---------|:--------:|:--------:|
| **81 字符串** | 纯文本 | 通用交换、API | ⭐⭐⭐⭐⭐ | ✅ 必须 |
| **SDM** | 纯文本 | 批量谜题库 | ⭐⭐⭐⭐ | ✅ 必须 |
| **SDK** | 纯文本 | 单谜题交换 | ⭐⭐⭐⭐ | ✅ 必须 |
| **SS** | 纯文本 | 可读性交换 | ⭐⭐⭐ | ✅ 推荐 |
| **OpenSudoku XML** | XML | Android 生态 | ⭐⭐⭐ | 🔜 可选 |
| **JSON** | JSON | 现代 API | ⭐⭐⭐⭐ | ✅ 推荐 |
| **种子 ID** | 自定义 | 本项目特有 | ⭐ (仅本项目) | ✅ 已支持 |

## 二、各格式详细规范

### 2.1 📄 81 字符串格式（最通用）

这是最简单、最广泛使用的格式，几乎所有数独软件和 API 都支持。

**规范：**
- 长度：恰好 81 个字符
- 已知数：`1`-`9`
- 空格：`0` 或 `.`（两种都应支持）
- 行序：从左到右，从上到下（第 1-9 个字符 = 第 1 行）
- 忽略：空格、换行、其他非数字非点字符

**示例：**
```
530070000600195000098000060800060003400803001700020006060000280000419005000080079
```

或用 `.` 表示空格：
```
53..7....6..195....98....6.8...6...34..8.3..17...2...6.6....28....419..5....8..79
```

**解析规则：**
```
输入 → 去除所有非 [0-9.] 字符 → 验证长度 == 81 → 解析为 9×9 棋盘
其中 '0' 和 '.' 均视为空格 (值 = 0)
```

**使用场景：**
- 剪贴板复制粘贴
- API 请求/响应
- 在线论坛谜题交流
- 数据库存储

---

### 2.2 📦 SDM 格式（Sudoku Data Manager）

SDM 是批量存储谜题的标准格式，广泛用于谜题库和谜题集。

**规范：**
- 每行一个谜题（81 字符串格式）
- 空格用 `0` 或 `.` 表示
- 可以有注释行（以 `#` 开头）
- 文件后缀通常为 `.sdm`

**示例：**
```
# Easy puzzles collection
# Generated: 2024-01-01
016400000200009000400000062070230100100000003003087040960000005000800007000006820
530070000600195000098000060800060003400803001700020006060000280000419005000080079
```

**解析规则：**
```
逐行读取 → 跳过 # 开头 → 跳过空行 → 每行作为一个 81 字符串解析
```

**使用场景：**
- 谜题集发布与分享
- Sudoku 10,000 等软件的谜题库格式
- 批量导入大量谜题
- 开源谜题数据库（如 Sudoku Exchange Puzzle Bank）

---

### 2.3 📝 SDK 格式（Sadman Sudoku）

SDK 是 Sadman Sudoku 定义的格式，增加了可读性和元数据支持。

**规范：**
- 9 行数字，每行 9 个字符
- 空格用 `.` 表示
- 可选元数据行（以 `#` 开头）：
  - `#A` — 作者 (Author)
  - `#D` — 描述 (Description)
  - `#C` — 注释 (Comment)
  - `#B` — 难度/评级 (Difficulty)
  - `#S` — 来源 (Source)
- 文件后缀通常为 `.sdk`

**示例：**
```
#D Weekly Challenge #42
#A Puzzle Master
#B Hard
#S SudokuWeekly.com
2..1.5..3
.54...71.
.1.2.3.8.
6.28.73.4
.........
1.53.98.6
.2.7.1.6.
.81...24.
7..4.2..1
```

**解析规则：**
```
逐行读取 → 以 # 开头的行解析为元数据 → 非#行拼接为 81 字符串 → 解析棋盘
```

**使用场景：**
- 单谜题文件分享
- 带作者/来源信息的谜题
- Sadman Sudoku、SudoCue 等软件

---

### 2.4 🔲 SS 格式（Simple Sudoku）

SS 格式由 Simple Sudoku 软件定义，强调人类可读性。

**规范：**
- 9 行，每行数字之间用空格分隔
- 空格用 `.` 表示
- 每 3 行之间可有空行分隔（可选）
- 每行内每 3 个数字之间可有额外空格（可选）
- 文件后缀通常为 `.ss`

**示例（紧凑版）：**
```
2 . . 1 . 5 . . 3
. 5 4 . . . 7 1 .
. 1 . 2 . 3 . 8 .
6 . 2 8 . 7 3 . 4
. . . . . . . . .
1 . 5 3 . 9 8 . 6
. 2 . 7 . 1 . 6 .
. 8 1 . . . 2 4 .
7 . . 4 . 2 . . 1
```

**示例（分组版）：**
```
2 . .  1 . 5  . . 3
. 5 4  . . .  7 1 .
. 1 .  2 . 3  . 8 .

6 . 2  8 . 7  3 . 4
. . .  . . .  . . .
1 . 5  3 . 9  8 . 6

. 2 .  7 . 1  . 6 .
. 8 1  . . .  2 4 .
7 . .  4 . 2  . . 1
```

**解析规则：**
```
逐行读取 → 去除空格 → 提取所有 [0-9.] 字符 → 拼接为 81 字符串 → 解析棋盘
```

**使用场景：**
- 人类可读的谜题展示
- 邮件/消息中分享谜题
- Simple Sudoku 软件

---

### 2.5 🌐 OpenSudoku XML 格式

OpenSudoku 是开源的 Android 数独应用，定义了 XML 格式用于谜题批量管理。

**规范：**
```xml
<?xml version="1.0" encoding="utf-8"?>
<opensudoku version="2">
  <game
    created="1234567890"
    state="0"
    time="0"
    last_played="0"
    data="530070000600195000098000060800060003400803001700020006060000280000419005000080079"
    note=""
    command_stack=""
  />
  <game
    created="1234567891"
    state="0"
    time="0"
    last_played="0"
    data="016400000200009000400000062070230100100000003003087040960000005000800007000006820"
    note=""
    command_stack=""
  />
</opensudoku>
```

**核心字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | 整数 | 格式版本（当前为 2） |
| `created` | 时间戳 | 创建时间 |
| `state` | 整数 | 0=未开始, 1=进行中, 2=已完成 |
| `time` | 毫秒 | 已用时间 |
| `data` | 字符串 | 81 字符谜题数据 |
| `note` | 字符串 | 笔记/候选数数据（可选） |
| `command_stack` | 字符串 | 撤销历史（可选） |

**解析规则：**
```
解析 XML → 遍历 <game> 元素 → 提取 data 属性 → 作为 81 字符串解析
```

**使用场景：**
- Android 生态谜题交换
- OpenSudoku App 用户的谜题导入/导出
- 批量谜题管理

### 2.7 🎲 种子 ID 格式（本项目已有）

本项目已实现的自有格式，用于谜题精确复现和分享。

**规范：**
```
格式: difficulty:symmetry:seed
示例: 3:rot180:123456789

difficulty: 1-5 (Easy/Medium/Hard/Expert/Evil)
symmetry: none | rot180 | rot90 | mirror
seed: 正整数（随机种子）
```

**特点：**
- ✅ 可精确复现谜题
- ✅ 极其紧凑
- ❌ 仅本项目可用
- ❌ 不携带棋盘数据本身

## 五、实现方案

### 5.1 导入功能设计

#### 5.1.1 支持的导入方式

```
导入入口
├── 📋 剪贴板粘贴（已有种子ID，扩展支持其他格式）
├── 📝 文本输入框（手动输入 81 字符串）
├── 📂 文件选择（.sdm / .sdk / .ss / .xml / .json）
└── 🌐 URL 导入（可选，从 API 获取）
```

#### 5.1.2 智能格式识别

导入时应自动识别格式，而非让用户手动选择：

```
输入文本
  │
  ├── 匹配 "difficulty:symmetry:seed" → 种子 ID
  │
  ├── 以 '<?xml' 或 '<opensudoku' 开头 → OpenSudoku XML
  │
  ├── 以 '{' 开头 → JSON 格式
  │
  ├── 包含空格分隔的数字行 → SS 格式
  │
  ├── 包含 9 行各 9 字符 → SDK 格式
  │
  └── 去除非数字非点字符后恰好 81 字符 → 81 字符串 / SDM
```

#### 5.1.3 解析器伪代码

```typescript
// 统一解析接口
interface ParsedPuzzle {
  board: number[][]           // 9×9 棋盘 (0=空格)
  metadata?: {
    author?: string
    description?: string
    difficulty?: string
    source?: string
  }
}

function parseSudokuInput(input: string): ParsedPuzzle[] {
  const trimmed = input.trim()

  // 1. 尝试种子 ID
  if (isSeedId(trimmed)) {
    return [generateFromSeed(trimmed)]
  }

  // 2. 尝试 XML
  if (trimmed.startsWith('<?xml') || trimmed.startsWith('<opensudoku')) {
    return parseOpenSudokuXml(trimmed)
  }

  // 3. 尝试 JSON
  if (trimmed.startsWith('{') || trimmed.startsWith('[')) {
    return parseJsonFormat(trimmed)
  }

  // 4. 尝试 SDK (带元数据的 9 行格式)
  if (hasSdkMetadata(trimmed)) {
    return [parseSdkFormat(trimmed)]
  }

  // 5. 尝试 SS (空格分隔格式)
  if (hasSsFormat(trimmed)) {
    return [parseSsFormat(trimmed)]
  }

  // 6. 尝试 SDM (多行 81 字符串)
  const lines = trimmed.split('\n')
    .filter(l => !l.startsWith('#') && l.trim().length > 0)
  if (lines.length > 1 && lines.every(l => extractDigits(l).length === 81)) {
    return lines.map(l => parse81String(l))
  }

  // 7. 降级到 81 字符串
  const digits = extractDigits(trimmed)
  if (digits.length === 81) {
    return [parse81String(digits)]
  }

  throw new Error('无法识别的数独格式')
}
```

### 5.2 导出功能设计

#### 5.2.1 支持的导出格式

| 格式 | 导出方式 | 适用场景 |
|------|---------|---------|
| 81 字符串 | 复制到剪贴板 | 通用分享 |
| SDK | 复制到剪贴板 | 带描述的分享 |
| SS | 复制到剪贴板 | 人类可读 |
| 种子 ID | 复制到剪贴板 | 本项目用户间分享 |
| JSON | 复制到剪贴板 | 开发者/API 对接 |

#### 5.2.2 导出生成器伪代码

```typescript
function exportPuzzle(board: number[][], format: string, metadata?: object): string {
  switch (format) {
    case '81string':
      // "530070000600195000..."
      return board.flat().map(v => v || '0').join('')

    case 'sdk':
      // 带元数据的 9 行格式
      let result = ''
      if (metadata?.description) result += `#D ${metadata.description}\n`
      if (metadata?.author) result += `#A ${metadata.author}\n`
      if (metadata?.difficulty) result += `#B ${metadata.difficulty}\n`
      for (const row of board) {
        result += row.map(v => v || '.').join('') + '\n'
      }
      return result

    case 'ss':
      // 空格分隔的人类可读格式
      return board.map(row =>
        row.map(v => v || '.').join(' ')
      ).join('\n')

    case 'json':
      return JSON.stringify({ puzzle: board, metadata }, null, 2)

    default:
      return board.flat().map(v => v || '0').join('')
  }
}
```

### 5.3 验证与错误处理

导入的谜题必须通过以下验证：

```
验证流程
  │
  ├── 1. 格式验证：是否符合格式规范
  │
  ├── 2. 数值验证：所有值在 0-9 范围内
  │
  ├── 3. 冲突验证：已知数是否有行/列/宫冲突
  │
  ├── 4. 可解验证：谜题是否有唯一解
  │     ├── 有唯一解 → ✅ 导入成功
  │     ├── 无解 → ❌ 提示"此谜题无解"
  │     └── 多解 → ⚠️ 提示"此谜题有多个解"（允许导入但标记）
  │
  └── 5. 难度评估：使用 DifficultyEvaluator 自动评级
```

```
        ├── parseJsonFormat()     # JSON 格式解析
        ├── exportTo81String()    # 导出为 81 字符串
        ├── exportToSdk()         # 导出为 SDK 格式
        ├── exportToSs()          # 导出为 SS 格式
        ├── exportToJson()        # 导出为 JSON 格式
        └── detectFormat()        # 格式自动检测
```

### 7.3 首页集成

在现有首页 `Index.ets` 的种子输入区域旁边添加 "导入谜题" 入口：

## 八、实现优先级

### Phase 1（必须支持 — 覆盖 90% 场景）

1. **81 字符串导入/导出** — 最通用，解析最简单
2. **SDK 格式导入** — 带元数据，常见于谜题社区
3. **SS 格式导入** — 人类可读，方便用户手动输入
4. **剪贴板智能识别** — 粘贴后自动判断格式

### Phase 2（推荐支持 — 扩大生态兼容性）

5. **SDM 批量导入** — 支持谜题集/谜题库导入
6. **JSON 格式** — 对接在线 API
7. **导出功能** — 支持多格式导出到剪贴板

### Phase 3（可选支持 — 特定生态）

8. **OpenSudoku XML** — Android 生态互通
9. **文件导入** — 本地文件选择
10. **URL 导入** — 从在线 API 获取
