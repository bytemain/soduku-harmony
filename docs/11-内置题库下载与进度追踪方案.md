# 📥 内置题库下载与进度追踪方案

> 设计内置的数独题库下载功能，让用户可以直接在 App 内浏览、下载、游玩外部题库，
> 并追踪每道题的完成进度。
>
> 参考：Doc 10 竞品分析 §四 P0/P1

---

## 一、现状与目标

### 1.1 当前能力

| 能力 | 现状 | 问题 |
|------|------|------|
| 题目来源 | 仅动态生成 / 手动粘贴导入 | 无法发现和游玩经典名题 |
| 导入格式 | 支持 6 种格式 (81/SDM/SDK/SS/JSON/Seed) | 已有格式解析能力，但无前端入口 |
| 进度管理 | 仅单局保存 (`GameStorage`) | 切换题目会覆盖之前的进度 |
| 题库浏览 | 无 | 用户需要自行搜索题目字符串 |

### 1.2 目标

1. **内置题库** — App 内预置精选题集，无需联网即可使用
2. **外部题库** — 支持从常见数独网站/资源下载题集
3. **进度追踪** — 每道题独立保存进度，支持断点续玩
4. **题库管理** — 浏览、筛选、排序、搜索题目

## 二、数据模型设计

### 2.1 题库结构

```
PuzzleLibrary (题库)
├── id: string                    // 唯一标识（如 "builtin-classic-easy"）
├── name: string                  // 显示名称
├── description: string           // 题库描述
├── source: string                // 来源（内置 / URL / 文件）
├── sourceUrl?: string            // 原始下载 URL
├── format: SudokuFormatType      // 原始格式
├── puzzleCount: number           // 题目总数
├── difficulty?: Difficulty       // 整体难度（可选）
├── tags: string[]                // 标签（如 ["classic", "hard", "17-clue"]）
├── createdAt: number             // 导入时间
├── updatedAt: number             // 最后更新时间
└── puzzles: PuzzleEntry[]        // 题目列表
```

### 2.2 题目条目

```
PuzzleEntry (单道题)
├── index: number                 // 在题库中的序号（0-based）
├── puzzle81: string              // 81 字符串格式的谜题
├── solution81?: string           // 81 字符串格式的答案（可选，延迟计算）
├── difficulty?: Difficulty       // 评估难度
├── metadata?: PuzzleMetadata     // 元数据（作者、描述等）
└── progress: PuzzleProgress      // 完成进度
```

### 2.3 进度数据

```
PuzzleProgress (进度)
├── status: 'new' | 'in_progress' | 'completed'
├── startedAt?: number            // 开始时间
├── completedAt?: number          // 完成时间
├── elapsedSeconds: number        // 累计用时
├── errorCount: number            // 错误次数
├── cellStates?: string           // 序列化的 cells 状态（断点续玩）
└── percentage: number            // 完成百分比 (0-100)
```

### 2.4 TypeScript 接口

```typescript
// entry/src/main/ets/models/PuzzleLibrary.ets

export type LibrarySource = 'builtin' | 'url' | 'file' | 'clipboard'

export type PuzzleStatus = 'new' | 'in_progress' | 'completed'

export interface PuzzleProgress {
  status: PuzzleStatus
  startedAt?: number
  completedAt?: number
  elapsedSeconds: number
  errorCount: number
  cellStates?: string
  percentage: number
}

export interface PuzzleEntry {
  index: number
  puzzle81: string
  solution81?: string
  difficulty?: number
  metadata?: PuzzleMetadata
  progress: PuzzleProgress
}

export interface PuzzleLibrary {
  id: string
  name: string
  description: string
  source: LibrarySource
  sourceUrl?: string
  format: string
  puzzleCount: number
  difficulty?: number
  tags: string[]
  createdAt: number
  updatedAt: number
}
```

| 来源 | 格式 | 示例 URL | 说明 |

| **手动导入** | 任意支持格式 | — | 从剪贴板/文件粘贴 |

### 4.2 已知开源数独题库

| 题库名称 | 格式 | 题数 | 说明 |
|---------|------|------|------|
| Gordon Royle 17-clue | 一行一题 | 49,158 | 已证明完备的 17 线索集合 |
| Magictour Top 1465 | 一行一题 | 1,465 | 经典困难题集 |
| Sudoku Exchange | SDM | 数千 | 社区贡献 |
| AI Escargot & Hard Puzzles | 81-string | 数十 | 著名困难谜题 |
| Project Euler #96 | Grid | 50 | 编程竞赛用题 |

### 4.3 下载流程

```
用户选择题库
    ↓
发起 HTTP 请求下载文件
    ↓
parseSudokuInput(content) 解析
    ↓
为每道题生成 PuzzleEntry
    ↓
评估难度（异步，逐题）
    ↓
存储到 PuzzleLibraryStorage
    ↓
显示在题库列表中
```

## 五、进度追踪系统

### 5.1 存储架构

```
Preferences 存储
├── puzzle_libraries          // JSON: PuzzleLibrary[] 索引
├── library_{id}_puzzles      // JSON: PuzzleEntry[] 每个题库的题目列表
├── library_{id}_progress     // JSON: PuzzleProgress[] 进度数据（频繁更新）
└── active_puzzle             // JSON: { libraryId, puzzleIndex } 当前活跃题目
```

### 5.2 进度同步

| 事件 | 操作 |
|------|------|
| 开始题目 | `status = 'in_progress'`, 记录 `startedAt` |
| 每次操作 | 更新 `cellStates` 和 `percentage` |
| 暂停/退出 | 保存 `elapsedSeconds` 和完整 `cellStates` |
| 完成 | `status = 'completed'`, 记录 `completedAt`、最终用时和错误数 |
| 重置 | 清除 `cellStates`，`status = 'new'` |

### 5.3 百分比计算

```typescript
function calculateProgress(cells: CellData[], totalCells: number): number {
  let filled = 0
  let givens = 0
  for (const cell of cells) {
    if (cell.isGiven) givens++
    else if (cell.value !== 0) filled++
  }
  const blanks = totalCells - givens
  if (blanks === 0) return 100
  return Math.round((filled / blanks) * 100)
}
```

## 七、技术实现要点

### 7.1 现有能力复用

| 现有模块 | 复用方式 |
|---------|---------|
| `SudokuFormat.parseSudokuInput()` | 解析下载/导入的题库文件 |
| `SudokuFormat.exportTo81String()` | 将题目标准化存储 |
| `DifficultyEvaluator` | 为题库中的题目评估难度 |
| `GameStorage` | 参考其序列化方案设计 `PuzzleProgress` |
| `PuzzleWorker` | 在 Worker 中异步处理题库解析和难度评估 |

### 7.2 存储方案

使用 HarmonyOS Preferences API（与现有 `GameStorage` 一致）：

```typescript
// PuzzleLibraryStorage.ets

export async function saveLibraryIndex(context: Context, libraries: PuzzleLibrary[]): Promise<void>
export function loadLibraryIndex(context: Context): PuzzleLibrary[]
export async function saveLibraryPuzzles(context: Context, libraryId: string, puzzles: PuzzleEntry[]): Promise<void>
export function loadLibraryPuzzles(context: Context, libraryId: string): PuzzleEntry[]
export async function saveProgress(context: Context, libraryId: string, puzzleIndex: number, progress: PuzzleProgress): Promise<void>

```

### 7.3 网络请求

使用 HarmonyOS `@ohos.net.http` 模块：

```typescript
import { http } from '@kit.NetworkKit'

async function downloadLibrary(url: string): Promise<string> {
  const httpRequest = http.createHttp()
  const response = await httpRequest.request(url, {
    method: http.RequestMethod.GET,
    expectDataType: http.HttpDataType.STRING,
  })
  httpRequest.destroy()
  return response.result as string
}
```
