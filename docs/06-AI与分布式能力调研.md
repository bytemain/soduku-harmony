## 2. Core Vision Kit — 拍照识别数独 📸

### 2.1 这是什么

Core Vision Kit 提供**通用文字识别（OCR）**能力，可以把图片中的印刷文字转为结构化数据。对数独 App 来说，最直接的用途是：

> **用户拍一张报纸/书上的数独照片 → App 自动识别成 9×9 棋盘数据 → 直接开始玩**

### 2.2 技术方案

```
拍照/选图 → PixelMap → Core Vision OCR → 文字+位置坐标 → 解析为 9×9 数组
```

#### 步骤 1：导入与初始化

```typescript
import { textRecognition } from '@kit.CoreVisionKit';
import { image } from '@kit.ImageKit';

// 页面加载时初始化 OCR 引擎
async aboutToAppear(): Promise<void> {
  await textRecognition.init();
}

// 页面销毁时释放
async aboutToDisappear(): Promise<void> {
  await textRecognition.release();
}
```

#### 步骤 2：获取图片并识别

```typescript
import { photoAccessHelper } from '@kit.MediaLibraryKit';
import { fileIo } from '@kit.CoreFileKit';

// 从图库选择图片
async selectAndRecognize() {
  let picker = new photoAccessHelper.PhotoViewPicker();
  let result = await picker.select({
    MIMEType: photoAccessHelper.PhotoViewMIMETypes.IMAGE_TYPE,
    maxSelectNumber: 1
  });

  let fileSource = await fileIo.open(result.photoUris[0], fileIo.OpenMode.READ_ONLY);
  let imageSource = image.createImageSource(fileSource.fd);
  let pixelMap = await imageSource.createPixelMap();

  // 调用 OCR
  let visionInfo: textRecognition.VisionInfo = { pixelMap: pixelMap };
  let config: textRecognition.TextRecognitionConfiguration = {
    isDirectionDetectionSupported: true   // 支持方向检测，防止照片歪了
  };

  let ocrResult = await textRecognition.recognizeText(visionInfo, config);
  // ocrResult.value = 识别出的全部文字
  // ocrResult.words = 每个字的位置、内容、置信度
  
  let board = this.parseSudokuFromOCR(ocrResult);
}
```

#### 步骤 3：从 OCR 结果解析数独棋盘

OCR 返回的 `TextRecognitionResult` 包含每个识别字符的**位置坐标**（bounding box），我们可以利用坐标还原 9×9 格局：

```typescript
function parseSudokuFromOCR(result: textRecognition.TextRecognitionResult): number[][] {
  let board: number[][] = Array.from({ length: 9 }, () => new Array(9).fill(0));

  // 1. 找出所有数字字符及其位置
  interface DigitInfo { digit: number; x: number; y: number; }
  let digits: DigitInfo[] = [];

  for (let word of result.words) {
    let text = word.text.trim();
    if (text.length === 1 && text >= '1' && text <= '9') {
      let loc = word.location;
      digits.push({
        digit: parseInt(text),
        x: loc.x + loc.width / 2,    // 中心 x
        y: loc.y + loc.height / 2     // 中心 y
      });
    }
  }

  // 2. 根据坐标聚类到 9×9 网格
  //    找到 x/y 的最大最小值，均匀分成 9 份
  if (digits.length === 0) return board;

  let minX = Math.min(...digits.map(d => d.x));
  let maxX = Math.max(...digits.map(d => d.x));
  let minY = Math.min(...digits.map(d => d.y));
  let maxY = Math.max(...digits.map(d => d.y));

  let cellW = (maxX - minX) / 8;   // 8 个间距 = 9 列
  let cellH = (maxY - minY) / 8;

  for (let d of digits) {
    let col = Math.round((d.x - minX) / cellW);
    let row = Math.round((d.y - minY) / cellH);
    col = Math.max(0, Math.min(8, col));
    row = Math.max(0, Math.min(8, row));
    board[row][col] = d.digit;
  }

  return board;
}
```

### 2.3 约束与注意

| 约束 | 说明 |
|------|------|
| 图片格式 | JPEG / JPG / PNG |
| 支持语言 | 中文、英文、日文、韩文 |
| 图片质量 | 建议 720p 以上，宽高 100px~15210px |
| 拍摄角度 | 镜头与纸面夹角 < 30° |
| 模拟器 | ❌ 不支持，必须真机 |
| 手写体 | 识别能力有限，印刷体最佳 |

## 3. MindSpore Lite Kit — 端侧 AI 推理 🧠

### 3.1 这是什么

MindSpore Lite 是华为的端侧 AI 推理框架，支持在设备上运行 `.ms` 格式的模型。**不需要联网，纯本地推理。**

```typescript
import { mindSporeLite } from '@kit.MindSporeLiteKit';
```

### 3.2 核心 API

| 接口 | 说明 |
|------|------|
| `loadModelFromBuffer(buffer, context)` | 从 buffer 加载模型 |
| `loadModelFromFile(path, context)` | 从文件加载模型 |
| `model.getInputs()` | 获取模型输入 tensor |
| `model.predict(inputs)` | 执行推理 |
| `tensor.getData()` | 读取 tensor 数据 |
| `tensor.setData(buffer)` | 设置 tensor 数据 |

### 3.3 在数独中的应用场景

#### 场景 A：智能难度评估

传统的难度评估靠「挖洞数 + 需要的解题技巧」来分级。可以训练一个小模型：

```
输入：81 个数字（0 = 空格）→ flatten 成长度 81 的 float 数组
输出：1 个 float（预测难度分数 0.0~1.0）

训练数据：大量已标注难度的数独题目
模型：几层全连接就够，很小（< 100KB）
```

```typescript
async function evaluateDifficulty(board: number[][]): Promise<number> {
  // 1. 准备上下文
  let context: mindSporeLite.Context = { target: ['cpu'] };
  context.cpu = { threadNum: 2, precisionMode: 'enforce_fp32' };

  // 2. 加载模型（从 rawfile）
  let modelBuffer = await getContext(this).resourceManager
    .getRawFileContent('sudoku_difficulty.ms');
  let model = await mindSporeLite.loadModelFromBuffer(modelBuffer.buffer, context);

  // 3. 准备输入（81 个 float32）
  let inputs = model.getInputs();
  let inputData = new Float32Array(81);
  for (let r = 0; r < 9; r++) {
    for (let c = 0; c < 9; c++) {
      inputData[r * 9 + c] = board[r][c] / 9.0;   // 归一化到 0~1
    }
  }
  inputs[0].setData(inputData.buffer);

  // 4. 推理
  let outputs = await model.predict(inputs);
  let outputData = new Float32Array(outputs[0].getData());
  return outputData[0];   // 0.0 = 极简单, 1.0 = 极困难
}
```

#### 场景 B：智能提示（下一步推荐）

比简单的「告诉你正确答案」更有教育意义——告诉用户「你应该看哪里、用什么技巧」：

```
输入：81 格当前状态 + 笔记状态
输出：推荐的格子 (row, col) + 推荐的技巧类型

技巧类型枚举：
  0 = Naked Single（裸单）
  1 = Hidden Single（隐式唯一）
  2 = Naked Pair（裸对）
  3 = Pointing Pair（指向对）
  4 = Box/Line Reduction
  ...
```

> 这个场景也可以不用 ML，用纯算法扫描技巧即可。ML 的优势是可以学习「人类觉得最自然的下一步」。

#### 场景 C：拍照识别增强

OCR 识别后，可以训练一个小的 CNN 分类器专门识别数字 1-9：

```
输入：单个格子的裁剪图像（28×28 灰度图）
输出：0~9 的分类概率

模型：类 MNIST 的小 CNN，< 200KB
```

当 Core Vision OCR 对某些字符识别信心不足时，用这个专用模型做二次确认。

### 3.4 系统能力配置

DevEco Studio 默认不包含 MindSpore Lite 系统能力，需要手动添加：

在 `entry/src/main/` 下创建 `syscap.json`：

```json
{
  "devices": {
    "general": ["default"]
  },
  "development": {
    "addedSysCaps": [
      "SystemCapability.AI.MindSporeLite"
    ]
  }

```


## 4. DataAugmentation Kit — 端侧大模型对话（AI 数独教练）🎓

### 4.1 这是什么

`@kit.DataAugmentationKit` 提供了 `localChatModel`——**端侧大模型对话能力**。完全在设备本地运行，不需要联网，支持多轮对话、流式输出。

> 这是鸿蒙系统内置的大语言模型，直接调用即可，不需要自己训练或部署模型。

### 4.2 核心 API

```typescript
import { localChatModel } from '@kit.DataAugmentationKit';
import { BusinessError } from '@kit.BasicServicesKit';

// 1. 初始化模型（页面加载时调用一次）
await localChatModel.init();

// 2. 发送对话
let question: localChatModel.QuestionInfo = {
  questionId: 1,
  content: '什么是 XY-Wing 技巧？请用简单的语言解释'
};

let config: localChatModel.Config = {
  isStream: true   // 流式输出，打字机效果
};

localChatModel.chat(question, config, (err: BusinessError, ans: localChatModel.Answer) => {
  if (err) {
    console.error(`Error: ${err.message}`);
    return;
  }
  if (ans.content) {
    // 逐段拼接流式输出
    assistantText += ans.content;
  }
  if (ans.isFinished) {
    // 回答结束
    console.log('Complete:', assistantText);
  }
});
```

#### 关键：把棋盘状态注入 Prompt

要让 AI 教练能分析当前局面，需要把棋盘序列化后作为上下文传给大模型：

```typescript
function buildCoachPrompt(board: number[][], notes: number[][]): string {
  // 将棋盘转为文本描述
  let boardStr = '';
  for (let r = 0; r < 9; r++) {
    boardStr += board[r].map(v => v === 0 ? '.' : `${v}`).join(' ') + '\n';
  }

  return `你是一位数独教练。以下是当前棋盘状态（. 表示空格）：

${boardStr}

请基于这个棋盘回答用户的问题。
规则：
1. 不要直接告诉答案，而是引导用户思考
2. 指出可以使用的解题技巧名称
3. 用简单易懂的语言解释
4. 如果涉及具体格子，使用 R行C列 的格式（如 R3C5 表示第3行第5列）`;
}

// 使用时
let systemPrompt = buildCoachPrompt(currentBoard, currentNotes);
let question: localChatModel.QuestionInfo = {
  questionId: chatCounter++,
```

## 6. 分布式键值数据库 — 跨设备数据同步 🔄

### 6.1 这是什么

`@kit.ArkData` 中的 `distributedKVStore` 模块，提供设备间的 KV 存储自动同步。

### 6.2 核心 API

```typescript
import { distributedKVStore } from '@kit.ArkData';

// 1. 创建 KVManager
const kvManagerConfig: distributedKVStore.KVManagerConfig = {
  context: getContext(this),
  bundleName: 'com.bytemain.soduku'
};
let kvManager = distributedKVStore.createKVManager(kvManagerConfig);

// 2. 获取 KVStore 实例
const options: distributedKVStore.Options = {
  createIfMissing: true,
  encrypt: false,
  backup: false,
  autoSync: true,                          // 自动同步到其他设备
  kvStoreType: distributedKVStore.KVStoreType.SINGLE_VERSION,
  securityLevel: distributedKVStore.SecurityLevel.S1
};
let kvStore = await kvManager.getKVStore<distributedKVStore.SingleKVStore>('sudoku_store', options);

// 3. 写入数据
await kvStore.put('currentGame', JSON.stringify(gameState));

// 4. 读取数据
let value = await kvStore.get('currentGame');

// 5. 监听数据变更（其他设备修改后自动触发）
kvStore.on('dataChange', distributedKVStore.SubscribeType.SUBSCRIBE_TYPE_ALL, (data) => {
  console.log('Data changed:', JSON.stringify(data));
  // 更新本地 UI
});
```

### 6.3 在数独中的应用：游戏进度无缝续接

```
场景：用户在手机上玩到一半 → 拿起平板 → 自动继续

实现：
  手机端：每次操作后 kvStore.put('game_state', serializedState)
  平板端：kvStore.on('dataChange') → 自动恢复到最新状态
  
同步数据结构：
{
  "board": [[...], ...],          // 81 格
  "solution": [[...], ...],       // 答案（用于验证）
```

## 7. 分布式数据对象 — 实时协同 👥

### 7.1 这是什么

`distributedDataObject` 提供**面向对象**的跨设备数据协同。与 KVStore 不同，它更适合**实时同步结构化状态**。

### 7.2 核心 API

```typescript
import { distributedDataObject } from '@kit.ArkData';

// 定义同步数据结构
class SudokuGameData {
  board: string = '';           // JSON 序列化的棋盘
  currentPlayer: string = '';   // 当前操作的玩家
  timestamp: number = 0;
}

// 创建分布式数据对象
let gameData = new SudokuGameData();
let dataObject = distributedDataObject.create(context, gameData);

// 生成并设置 sessionId（两端必须用同一个 sessionId）
let sessionId = distributedDataObject.genSessionId();
dataObject.setSessionId(sessionId);

// 监听数据变更
dataObject.on('change', (sessionId: string, fields: string[]) => {
  console.log('Fields changed:', fields);
  // 字段级别的变更通知！
});

// 监听对端上下线
dataObject.on('status', (sessionId: string, networkId: string, status: string) => {
  // status: 'online' | 'offline' | 'restored'
});

// 修改数据（会自动同步到对端）
dataObject['board'] = JSON.stringify(newBoard);
dataObject['currentPlayer'] = 'player2';

// 保存数据到远端设备
dataObject.save(remoteDeviceId);
```

## 8. MultimodalAwareness Kit — 左右手检测 🤚

### 8.1 这是什么

`@kit.MultimodalAwarenessKit` 的 `motion` 模块可以检测用户**当前用左手还是右手操作屏幕**，以及**哪只手握持设备**。

| 能力 | API 版本 | 说明 |
|------|----------|------|
| **操作手检测** | API 15+ | 检测触屏的是左手还是右手 |
| **握持手检测** | API 20+ | 检测握持设备的是左手还是右手 |

### 8.2 核心 API

```typescript
import { motion } from '@kit.MultimodalAwarenessKit';
import { BusinessError } from '@kit.BasicServicesKit';

// 一次性获取当前操作手
try {
  let status = motion.getOperatingHandStatus();
  console.log(`当前操作手: ${status.hand}`);  // 'left' | 'right'
} catch (err) {
  let e = err as BusinessError;
  console.error(`Code: ${e.code}, Message: ${e.message}`);
}

// 实时监听操作手切换
let callback = (data: motion.OperatingHandStatus) => {
  console.log(`操作手变了: ${data.hand}`);
  // 动态调整 UI 布局
};
motion.on('operatingHandChanged', callback);

// 取消监听
motion.off('operatingHandChanged', callback);
```

### 8.3 在数独中的应用：单手友好布局

#### 问题

3×3 手势键盘在屏幕底部居中，单手操作时：
- **右手用户**：左上角的按键（1）很难够到
- **左手用户**：右上角的按键（3）很难够到

#### 方案：根据操作手自动调整布局

```
右手模式（默认）：                 左手模式：
九键偏右 + 略微倾斜                九键偏左 + 镜像倾斜

┌──────────────────────┐      ┌──────────────────────┐
│                      │      │                      │
│      ┌───┬───┬───┐   │      │   ┌───┬───┬───┐      │
│      │ 1 │ 2 │ 3 │   │      │   │ 1 │ 2 │ 3 │      │
│      ├───┼───┼───┤   │      │   ├───┼───┼───┤      │
│      │ 4 │ 5 │ 6 │   │      │   │ 4 │ 5 │ 6 │      │
│      ├───┼───┼───┤   │      │   ├───┼───┼───┤      │
│      │ 7 │ 8 │ 9 │   │      │   │ 7 │ 8 │ 9 │      │
│      └───┴───┴───┘   │      │   └───┴───┴───┘      │
│              ← 偏右手 │      │ 偏左手 →              │
└──────────────────────┘      └──────────────────────┘
```

#### 还可以做什么

| 适配项 | 右手模式 | 左手模式 |
|--------|----------|----------|
| 九键位置 | 偏右对齐 | 偏左对齐 |
| 工具栏顺序 | 常用按钮靠右 | 常用按钮靠左 |
| 教练按钮🤖 | 右上角 | 左上角 |
| 撤销按钮 | 右侧 | 左侧 |

#### 实现思路

```typescript
import { motion } from '@kit.MultimodalAwarenessKit';

@Entry
@Component
struct GamePage {
  @State isLeftHand: boolean = false;

  aboutToAppear() {
    // 初始检测
    try {
      let status = motion.getOperatingHandStatus();
      this.isLeftHand = (status.hand === 'left');
    } catch (e) {
      // 设备不支持，默认右手
      this.isLeftHand = false;
    }

    // 实时监听切换
    motion.on('operatingHandChanged', (data) => {
      this.isLeftHand = (data.hand === 'left');
    });
  }

  build() {
    Column() {
      // ...

      // 工具栏：根据操作手调整按钮顺序
      Row() {
        if (this.isLeftHand) {
          // 左手模式：常用按钮放左边
          ToolButton({ icon: '💡', label: '提示' })
          ToolButton({ icon: '🗑️', label: '擦除' })
          ToolButton({ icon: '↩️', label: '撤销' })
          ToolButton({ icon: '⚙️', label: '设置' })
        } else {
          // 右手模式：常用按钮放右边
          ToolButton({ icon: '⚙️', label: '设置' })
          ToolButton({ icon: '↩️', label: '撤销' })
          ToolButton({ icon: '🗑️', label: '擦除' })
          ToolButton({ icon: '💡', label: '提示' })
        }
      }

      // 九键：根据操作手调整对齐方式
      GestureNumberPad()
        .alignSelf(this.isLeftHand ? ItemAlign.Start : ItemAlign.End)
        .margin(this.isLeftHand ? { left: 16 } : { right: 16 })
    }
  }
}
```

### 8.5 约束

- 设备不支持时返回错误码 `801`，需要 fallback 到默认右手模式
- 屏幕边缘 8mm 内的触摸不参与判断
- 多指同时操作时无法检测
- 戴手套会降低准确率

```json5

  "requestPermissions": [

    },
    {
      "name": "ohos.permission.READ_MEDIA",              // 读取相册（OCR选图）
      "reason": "用于从相册选取数独照片进行识别"
    },
    {
      "name": "ohos.permission.ACTIVITY_MOTION",         // 操作手检测
      "reason": "用于检测左右手操作以优化按键布局"
    },
    {
      "name": "ohos.permission.DETECT_GESTURE",          // 握持手检测
      "reason": "用于检测握持方式以优化单手操作体验"
    }
  ]
}
```

> Core Vision Kit（OCR）和 MindSpore Lite 不需要额外权限，它们是纯本地计算。
> 分布式能力需要 `DISTRIBUTED_DATASYNC` 权限，且用户需在设置中开启设备间信任。
> Motion 检测需要 `ACTIVITY_MOTION` + `DETECT_GESTURE`，设备不支持时静默 fallback。
