# 一、主页（HomePage）

## 1. 主页整体结构

主页是用户启动 App 后看到的第一个页面，负责：选择难度、开始新游戏、继续上次游戏、查看统计。

采用**纵向居中**布局，干净利落——标题 + 一个大按钮 + 难度选择器，没有多余的卡片：

```
┌──────────────────────────────────┐
│                                  │
│                                  │
│            🔢                    │
│           数 独                   │
│         S U D O K U              │
│                                  │
│                                  │
│  ┌──────────────────────────┐    │
│  │                          │    │
│  │     ▶  开始游戏           │    │  ← 主按钮（大而醒目）
│  │                          │    │
│  └──────────────────────────┘    │
│                                  │
│  ┌──────────────────────────┐    │
│  │                          │    │
│  │     ▶  继续游戏  00:23    │    │  ← 有存档时显示
│  │       困难 · 47%          │    │
│  └──────────────────────────┘    │
│                                  │
│  ┌──┬────┬────┬────┬──┐         │
│  │简│ 中 │ 难 │ 专 │地│         │  ← 分段按钮选难度
│  │单│ 等 │ 困 │ 家 │狱│         │
│  └──┴────┴────┴────┴──┘         │
│                                  │
│                                  │
│       📊 统计     ⚙️ 设置        │  ← 底部入口
│                                  │
└──────────────────────────────────┘
```

## 2. 主页各区域详细设计

### ① App 标题区

页面上部居中，展示 App 名称和图标，营造品牌感。

```
┌──────────────────────────────────┐
│                                  │
│              🔢                  │
│             数 独                 │  ← 中文名，大号字
│           S U D O K U            │  ← 英文名，字母间距加大
│                                  │
└──────────────────────────────────┘
```

| 元素 | 样式 |
|------|------|
| 图标 | App icon 或 emoji `🔢`，48vp |
| 中文名 | "数 独"，32vp，FontWeight.Bold，`#212121` |
| 英文名 | "S U D O K U"，14vp，字间距 4vp，`#9E9E9E` |
| 整体间距 | 图标与中文名 12vp，中文名与英文名 4vp |

---

### ② 开始游戏按钮

页面视觉中心，最醒目的操作入口。点击后以**当前选中的难度**生成新题并开始游戏。

```
┌──────────────────────────────────────┐
│                                      │
│           ▶  开始游戏                 │
│                                      │
└──────────────────────────────────────┘
```

| 属性 | 值 |
|------|------|
| 宽度 | `100% - 48vp`（左右各 24vp 边距） |
| 高度 | 52vp |
| 圆角 | 26vp（全圆角胶囊形） |
| 背景 | `#1976D2`（Material Blue 700） |
| 文字 | "▶  开始游戏"，18vp，FontWeight.Medium，`#FFFFFF` |
| 按压态 | 背景变深 `#1565C0` + scale(0.97)，150ms |
| 左右边距 | 24vp |

**点击行为**：

```
点击"开始游戏"
  │
  ├── 检查是否有存档
  │     ├── 无存档 → 以当前选中难度生成新题 → 导航到 GamePage
  │     └── 有存档 且 存档难度 === 选中难度 → 直接开始（覆盖？弹窗确认）
  │     └── 有存档 且 存档难度 !== 选中难度 → 弹出确认弹窗
  │           ├── "放弃旧游戏" → 删除存档 → 生成新题 → 导航
  │           └── "取消" → 无操作
  │
  └── 导航到 GamePage，传入 mode='new' + difficulty
```

### ③ 继续游戏按钮（条件显示）

**仅当存在未完成的存档时显示**，出现在"开始游戏"按钮下方，样式为次要按钮（描边样式），不抢主按钮风头。

```
┌──────────────────────────────────────┐
│  ▶  继续游戏                  00:23  │
│     困难 · 已完成 47%                │
└──────────────────────────────────────┘
```

| 属性 | 值 |
|------|------|
| 宽度 | 与"开始游戏"同宽 |
| 高度 | 52vp |
| 圆角 | 26vp（胶囊形） |
| 背景 | `#FFFFFF` |
| 边框 | 1.5vp solid `#1976D2` |
| 文字"继续游戏" | 16vp，FontWeight.Medium，`#1976D2`，左侧 |
| 用时 | 14vp，`#757575`，右对齐 |
| 副文字 | 13vp，`#9E9E9E`，"困难 · 已完成 47%" |
| 按压态 | 背景 `#E3F2FD`，150ms |
| 与主按钮间距 | 12vp |

**内部布局**：两行，第一行左"▶ 继续游戏" + 右"00:23"，第二行左"困难 · 已完成 47%"。

```
Row（两端对齐）
├── Column（左侧）
│   ├── "▶  继续游戏"
│   └── "困难 · 已完成 47%"
└── Text "00:23"（右侧，垂直居中）
```

**点击行为**：直接导航到 GamePage，传入 mode='continue' + savedState。

**长按行为**：弹出 ActionSheet 确认"放弃当前游戏？"→ 确认后删除存档，按钮消失。

### ④ 难度选择器（SegmentButton 分段按钮）

使用 HarmonyOS 的**分段按钮**组件（`SegmentButton`），一行展示 5 个难度，轻触即切换。选中状态醒目，交互零成本。

```
┌────┬────┬────┬────┬────┐
│ 简 │ 中 │ 困 │ 专 │ 地 │
│ 单 │ 等 │ 难 │ 家 │ 狱 │
└────┴────┴────┴────┴────┘
  🟢   🔵   🟡   🟠   🔴     ← 底部颜色指示条（选中项）
```

| 属性 | 值 |
|------|------|
| 布局 | `Row`，5 个等宽按钮，无间距 |
| 宽度 | `100% - 48vp` |
| 高度 | 44vp |
| 圆角 | 22vp（整体胶囊形） |
| 背景 | `#E8E8E8`（未选中区域底色） |
| 选中项背景 | `#FFFFFF`（白色凸起滑块） |
| 选中项阴影 | 微阴影，radius 4，`rgba(0,0,0,0.1)` |
| 选中项底部 | 3vp 圆形色条，颜色 = 难度色 |
| 未选中文字 | 14vp，`#616161` |
| 选中文字 | 14vp，FontWeight.Medium，`#212121` |
| 左右边距 | 24vp |
| 与按钮间距 | 20vp |
| 切换动画 | 滑块平移，200ms，Curve.EaseInOut |

**默认选中**：上次玩的难度（从本地存储读取），首次使用默认"中等"。

#### 难度色彩映射

| 难度 | 中文 | 标识色 | 选中底部色条 |
|------|------|--------|-------------|
| EASY | 简单 | `#4CAF50` 绿 | `#4CAF50` |
| MEDIUM | 中等 | `#2196F3` 蓝 | `#2196F3` |
| HARD | 困难 | `#FF9800` 橙 | `#FF9800` |
| EXPERT | 专家 | `#F57C00` 深橙 | `#F57C00` |
| EVIL | 地狱 | `#F44336` 红 | `#F44336` |

#### 实现方式

使用 HarmonyOS 系统组件 `SegmentButton`（分段按钮，API 11+）：

```typescript
// 分段按钮：系统组件，自带滑块切换动画
@State selectedDifficulty: number = 1  // 默认"中等"

SegmentButton({
  options: SegmentButtonOptions.capsule({
    buttons: [
      { text: '简单' },
      { text: '中等' },
      { text: '困难' },
      { text: '专家' },
      { text: '地狱' },
    ] as SegmentButtonItemTuple
  }),
  selectedIndexes: [this.selectedDifficulty],
})
.width('100%')
.height(44)
```

> 如果系统 `SegmentButton` 样式不够灵活（比如想加底部颜色条），可以自定义实现：5 个等宽 `Stack` + 白色滑块 `Column` 做动画平移。

## 4. 主页布局代码结构

```typescript
@Entry
@Component
struct HomePage {
  @StorageLink('hasSavedGame') hasSavedGame: boolean = false
  @StorageLink('savedDifficulty') savedDifficulty: string = ''
  @StorageLink('savedTime') savedTime: string = '00:00'
  @StorageLink('savedProgress') savedProgress: number = 0
  @StorageLink('lastDifficulty') lastDifficulty: number = 1  // 上次选的难度，默认中等

  @State selectedDifficulty: number = 1   // 分段按钮选中索引

  private difficultyNames: string[] = ['简单', '中等', '困难', '专家', '地狱']

  aboutToAppear() {
    this.selectedDifficulty = this.lastDifficulty
  }

  build() {
    Column() {

      // 弹性空白
      Blank().layoutWeight(3)

      // ① App 标题
      Column({ space: 4 }) {
        Text('🔢')
          .fontSize(48)
        Text('数 独')
          .fontSize(32)
          .fontWeight(FontWeight.Bold)
          .fontColor('#212121')
        Text('S U D O K U')
          .fontSize(14)
          .fontColor('#9E9E9E')
          .letterSpacing(4)
      }
      .alignItems(HorizontalAlign.Center)

      // 间距
      Blank().layoutWeight(2)

      // ② 开始游戏 主按钮
      Button('▶  开始游戏')
        .type(ButtonType.Capsule)
        .width('100%')
        .height(52)
        .fontSize(18)
        .fontWeight(FontWeight.Medium)
        .backgroundColor('#1976D2')
        .fontColor('#FFFFFF')
        .onClick(() => this.startNewGame())

      // ③ 继续游戏 次按钮（有存档时显示）
      if (this.hasSavedGame) {
        Button() {
          Row() {
            Column({ space: 2 }) {
              Text('▶  继续游戏')
                .fontSize(16)
                .fontWeight(FontWeight.Medium)
                .fontColor('#1976D2')
              Text(`${this.savedDifficulty} · 已完成 ${this.savedProgress}%`)
                .fontSize(13)
                .fontColor('#9E9E9E')
            }
            .alignItems(HorizontalAlign.Start)
            .layoutWeight(1)

            Text(this.savedTime)
              .fontSize(14)
              .fontColor('#757575')
          }
          .width('100%')
          .padding({ left: 20, right: 20 })
        }
        .type(ButtonType.Capsule)
        .width('100%')
        .height(52)
        .backgroundColor('#FFFFFF')
        .borderWidth(1.5)
        .borderColor('#1976D2')
        .margin({ top: 12 })
        .onClick(() => this.continueGame())
      }

      // 间距
      Blank().height(20)

      // ④ 难度分段按钮
      SegmentButton({
        options: SegmentButtonOptions.capsule({
          buttons: this.difficultyNames.map(name => ({ text: name }))
            as SegmentButtonItemTuple
        }),
        selectedIndexes: [this.selectedDifficulty],
      })
      .width('100%')
      .height(44)

      // 弹性空白
      Blank().layoutWeight(3)

      // ⑤ 底部功能入口
      Row() {
        BottomEntry({ icon: '📊', label: '统计' })
        BottomEntry({ icon: '⚙️', label: '设置' })
      }
      .width('100%')
      .justifyContent(FlexAlign.SpaceEvenly)
      .padding({ bottom: 24 })

    }
    .width('100%')
    .height('100%')
    .padding({ left: 24, right: 24 })
    .backgroundColor('#F5F5F5')
  }
}
```

## 5. 主页交互流程

### 5.1 启动 App

```
App 启动
  │
  ├── 读取上次难度 → 设置分段按钮默认选中
  ├── 检查本地存档 → 有 → hasSavedGame = true，显示"继续游戏"
  │                → 无 → hasSavedGame = false，隐藏
  │
  └── 显示主页
```

### 5.2 切换难度

```
点击分段按钮的某一段
  │
  └── selectedDifficulty 更新 → 记入本地存储（lastDifficulty）
      → "开始游戏"按钮始终可用，使用当前选中难度
```

> 切换难度不会影响已存在的存档，只影响"开始游戏"按钮生成的新题。

### 5.3 点击"开始游戏"

```
点击"开始游戏"
  │
  ├── 检查是否有存档
  │     ├── 无存档 → 以当前难度生成新题 → 导航到 GamePage
  │     └── 有存档 → 弹出 AlertDialog
  │           │  "当前有未完成的游戏（困难 · 47%），开始新游戏？"
  │           │
  │           ├── [继续旧游戏] → 导航到 GamePage，恢复存档
  │           ├── [开始新游戏] → 删除存档 → 生成新题 → 导航
  │           └── [取消] → 关闭弹窗
  │
  └── 导航到 GamePage，传入 mode='new' + difficulty
```

### 5.4 点击"继续游戏"

```
点击"继续游戏"
  │
  └── 从本地读取存档 → 导航到 GamePage，传入 mode='continue' + savedState
```

### 5.5 从 GamePage 返回

```
GamePage 返回主页
  │
  ├── 游戏未完成 → 自动保存存档 → 主页显示"继续游戏"
  └── 游戏已完成 → 清除存档 → 更新统计数据 → 主页隐藏"继续游戏"
```

## 2. 各区域详细设计

### ① 顶部信息栏

```
┌──────────────────────────────────────┐
│  困难           00:23           ✕ 1/3 │
│  (难度)        (计时器)      (错误次数) │
└──────────────────────────────────────┘
```

- **左侧**：当前难度标签
- **中间**：计时器 `MM:SS` 格式
- **右侧**：错误次数 `✕ 当前/上限`（如 3 次机会）

**布局**：`Row` + `justifyContent(FlexAlign.SpaceBetween)`

```
高度：约 48vp
左右内边距：16vp
```

### 手势模式提示条（Gesture Mode Indicator）

位于**棋盘底部与操作区之间**的狭长提示条，仅在手势键盘**滑动过程中**显示，松手后自动消失。**不覆盖棋盘，不遮挡任何格子**。

#### 位置与布局

```
┌──────────────────────────────────┐
│                                  │
│           数独棋盘                │
│                                  │
├──────────────────────────────────┤  ← 棋盘底边
│  ✏️  笔记模式                     │  ← 提示条（棋盘与操作区之间，28vp 高）
├──────────────────────────────────┤  ← 操作区顶边
│           操作区                  │
│                                  │
```

**关键原则**：提示条占据棋盘和操作区之间的间隙空间，是独立的一行，绝不叠加在棋盘之上。

#### 显示规则

| 手势状态 | 提示条内容 | 背景色 | 图标 |
|----------|-----------|--------|------|
| 无滑动 / 点击 | **隐藏**（高度为 0，不占空间） | — | — |
| 上滑中（笔记） | `✏️ 笔记模式` | `#E3F2FD`（浅蓝） | ✏️ |
| 右滑中（删除线） | `～～ 删除线模式` | `#FFF3E0`（浅橙） | ～～ |
| 左滑中（擦除） | `🧹 擦除笔记` | `#F5F5F5`（浅灰） | 🧹 |
| 下滑中（预留） | `⬇️ 预留功能` | `#F5F5F5`（浅灰） | ⬇️ |
| 松手 | 淡出消失（200ms） | — | — |

#### 视觉样式

```
高度：28vp（显示时）/ 0vp（隐藏时，通过动画过渡）
宽度：100%（与棋盘同宽）
左侧内边距：16vp
圆角：0（与棋盘/操作区衔接，不需要圆角）
文字：13vp，FontWeight.Medium
图标 + 文字水平排列，垂直居中
动画：显示/隐藏使用 animateTo，duration 150ms，曲线 Curve.EaseOut
```

#### 交互时序

```
手指按下数字键 → 无变化（提示条隐藏）
         │
手指滑动超过阈值 → 识别方向 → 提示条淡入 + 展开（0vp → 28vp）
         │                      显示对应模式文字和图标
         │
手指继续滑动 → 如果方向改变 → 提示条内容切换（交叉淡入淡出）
         │
手指松开 → 执行操作 → 提示条淡出 + 收起（28vp → 0vp，200ms）
```

## 6. 3×3 手势键盘详细设计

### 核心理念

用 **3×3 九宫格** 布局代替传统数字行，每个键支持 **点击 + 四方向滑动**，让一根手指完成「填数 / 笔记 / 删除线 / 擦除」所有操作，不需要切换模式。

### 手势映射

```
                  ▲ 上滑
                  │ 笔记（toggle ACTIVE）
                  │
   左滑 ◄─────── ● ────────► 右滑
   擦除笔记       点击        删除线（toggle ELIMINATED）
   (→EMPTY)      填数/擦数    
                  │
                  │
                  ▼ 下滑
                  预留（可扩展）
```

| 手势 | 操作 | 对选中格的效果 |
|------|------|----------------|
| **点击** | 填数 | `cell.value = digit`；如果已有相同数字则擦除（toggle） |
| **上滑** | 笔记 | 切换 `notes` 第 d 位：`EMPTY ↔ ACTIVE`；如果格子有 value 则先清空 value |
| **右滑** | 删除线 | 切换 `notes` 第 d 位：`EMPTY/ACTIVE → ELIMINATED`，`ELIMINATED → EMPTY` |
| **左滑** | 擦除笔记 | 将 `notes` 第 d 位设为 `EMPTY`（无论当前是 ACTIVE 还是 ELIMINATED） |
| **下滑** | 预留 | 暂不实现，可后续绑定功能（如"填入并锁定"） |

### 手势判定逻辑

```typescript
// 手势方向判定
function resolveGesture(dx: number, dy: number, threshold: number): GestureAction {
  const dist = Math.sqrt(dx * dx + dy * dy)

  if (dist < threshold) {
    return GestureAction.TAP           // 位移不够 → 视为点击
  }

  // 计算角度（弧度转角度）
  const angle = Math.atan2(-dy, dx) * 180 / Math.PI
  // angle: 右=0°, 上=90°, 左=±180°, 下=-90°

  if (angle > 45 && angle <= 135) {
    return GestureAction.SWIPE_UP      // 笔记
  } else if (angle > -45 && angle <= 45) {
    return GestureAction.SWIPE_RIGHT   // 删除线
  } else if (angle > -135 && angle <= -45) {
    return GestureAction.SWIPE_DOWN    // 预留
  } else {
    return GestureAction.SWIPE_LEFT    // 擦除笔记
  }
}

enum GestureAction {
  TAP,
  SWIPE_UP,
  SWIPE_RIGHT,
  SWIPE_DOWN,
  SWIPE_LEFT,
}
```

> `threshold` 建议 **30vp**，太小容易误触，太大滑着累。

### ArkUI 手势实现

```typescript
@Component
struct GestureKey {
  @Prop digit: number
  @Prop remaining: number        // 剩余计数
  @State pressing: boolean = false
  @State swipeDir: string = ''   // 用于视觉反馈：'up'|'right'|'down'|'left'|''

  private startX: number = 0
  private startY: number = 0
  private readonly THRESHOLD = 30

  build() {
    Column() {
      Text(`${this.digit}`)
        .fontSize(28)
        .fontWeight(FontWeight.Medium)
        .fontColor(this.remaining > 0 ? '#212121' : '#9E9E9E')

      Text(`${this.remaining}`)
        .fontSize(11)
        .fontColor('#9E9E9E')
    }
    .width('100%')
    .height('100%')
    .justifyContent(FlexAlign.Center)
    .borderRadius(12)
    .backgroundColor(this.getKeyBg())
    .opacity(this.remaining > 0 ? 1.0 : 0.5)
    .scale({ x: this.pressing ? 0.92 : 1.0, y: this.pressing ? 0.92 : 1.0 })
    .animation({ duration: 100 })
    .gesture(
      PanGesture({ fingers: 1, distance: 1 })    // distance=1 让它几乎立刻触发
        .onActionStart((event: GestureEvent) => {
          this.pressing = true
          this.startX = event.offsetX
          this.startY = event.offsetY
        })
        .onActionUpdate((event: GestureEvent) => {
          // 实时计算方向，更新视觉指示 + 通知页面级提示条
          const dx = event.offsetX - this.startX
          const dy = event.offsetY - this.startY
          this.swipeDir = this.calcDir(dx, dy)
          this.onSwipeDirectionChange(this.swipeDir)  // 回调 → 驱动提示条显示
        })
        .onActionEnd((event: GestureEvent) => {
          this.pressing = false
          const dx = event.offsetX - this.startX
          const dy = event.offsetY - this.startY
          const action = resolveGesture(dx, dy, this.THRESHOLD)
          this.handleAction(action)
          this.swipeDir = ''
          this.onSwipeDirectionChange('')             // 回调 → 驱动提示条隐藏
        })
    )
  }
}
```

### 视觉反馈

滑动过程中，有两处视觉反馈：

**① 按键本身**：给出方向缩放提示

| 状态 | 按键视觉效果 |
|------|----------|
| 按下未移动 | 按键缩小（scale 0.92）+ 背景变深 |
| 上滑中 | 背景变为浅蓝 |
| 右滑中 | 背景变为浅橙 |
| 左滑中 | 背景变为浅灰 |
| 松手 | 恢复原状，执行对应操作 |

**② 手势模式提示条**（棋盘下方）：告诉用户当前滑动将执行什么操作

| 状态 | 提示条效果 |
|------|----------|
| 上滑中 | `✏️ 笔记模式`，浅蓝底 `#E3F2FD` |
| 右滑中 | `～～ 删除线模式`，浅橙底 `#FFF3E0` |
| 左滑中 | `🧹 擦除笔记`，浅灰底 `#F5F5F5` |
| 松手 | 提示条淡出消失（150ms） |

> 提示条不遮挡棋盘，它位于棋盘和操作区之间的独立空间，通过高度动画（0 ↔ 28vp）展开/收起。

## 7. 交互流程

### 7.1 手势键盘操作流程

```
┌─────────────┐     ┌──────────────┐     ┌────────────────────────────────────┐
│ 点击棋盘格子  │ ──→ │  格子被选中    │ ──→ │ 在手势键盘上操作数字 d              │
│ (row, col)   │     │  高亮同行/列/宫│     │                                    │
└─────────────┘     └──────────────┘     └──────┬──────┬──────┬──────┬───────┘
                                                 │      │      │      │
                                              点击    上滑    右滑    左滑
                                              填数    笔记    删除线  擦笔记
                                                 │      │      │      │
                                                 ▼      ▼      ▼      ▼
                                          ┌──────────┐ ┌────────┐ ┌──────────┐ ┌────────┐
                                          │value = d │ │notes d │ │notes d   │ │notes d │
                                          │重复则擦除 │ │EMPTY↔  │ │→ELIMINATED│ │→EMPTY  │
                                          │检查冲突  │ │ACTIVE  │ │ELIM→EMPTY│ │        │
                                          │检查完成  │ │        │ │          │ │        │
                                          └──────────┘ └────────┘ └──────────┘ └────────┘
```

### 7.2 点击（填数）的详细逻辑

```
if 选中格是 GIVEN → 无操作
if 选中格.value === digit → 擦除（value = 0）
else → 填入（value = digit，清空 notes）
    → 检查行/列/宫冲突 → 冲突则标记 ERROR flag
    → 检查棋盘是否完成
```

### 7.3 上滑（笔记）的详细逻辑

```
if 选中格是 GIVEN → 无操作
if 选中格有 value → 先清空 value（变为空格）
getNoteState(notes, digit):
  EMPTY   → 设为 ACTIVE
  ACTIVE  → 设为 EMPTY
  ELIMINATED → 设为 ACTIVE（覆盖删除线，恢复为正常笔记）
```

### 7.4 右滑（删除线）的详细逻辑

```
if 选中格是 GIVEN → 无操作
getNoteState(notes, digit):
  EMPTY       → 设为 ELIMINATED（直接标删除线）
  ACTIVE      → 设为 ELIMINATED（从笔记变为删除线）
  ELIMINATED  → 设为 EMPTY（取消删除线）
```

### 7.5 左滑（擦除笔记）的详细逻辑

```
getNoteState(notes, digit):
  任何状态 → 设为 EMPTY（简单粗暴清掉该位）
```

### 7.6 选中格的切换

```
点击已选中的格子 → 取消选中
点击另一个格子   → 切换选中到新格子
点击给定格      → 只选中+高亮，不允许修改
```

### 7.7 工具栏擦除

```
选中格有用户填入的数字 → 清空 value，恢复为 0
选中格有笔记          → 清空所有 notes（整个 notes = 0）
选中格是给定数字      → 无操作
```

> 工具栏"擦除"是全清操作，和左滑"擦除单个笔记"不同。
