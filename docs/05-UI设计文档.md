# 一、主页（HomePage）

## 1. 整体结构

纵向居中布局，从上到下 5 个区域。简洁、聚焦——打开 App 第一眼就知道该做什么。

```
│  │  ▶ 继续游戏    00:23      │    │    ③ 继续按钮（有存档时才显示）
```

## 2. 各区域详细设计

### ① App 标题区

简单但有辨识度：图标 + 中文大名 + 英文小名。

| 元素 | 样式 |
|------|------|
| 图标 🔢 | 48vp |
| 中文名 "数 独" | 32vp, Bold, `#212121`，字间距 8vp |
| 英文名 "S U D O K U" | 14vp, `#9E9E9E`，字间距 4vp |

**设计理由**：中英文双名增加品质感。英文名用字间距拉开，视觉上更"设计"。

### ② 开始游戏按钮

整个主页最显眼的元素——胶囊形大按钮。

| 属性 | 值 |
|------|------|
| 尺寸 | 宽 100%-48vp（即屏幕宽减两侧 24vp 边距），高 52vp |
| 圆角 | 26vp（胶囊形 = 高度的一半） |
| 背景 | `#1976D2`（Material Blue 700） |
| 按压态 | 背景变 `#1565C0` + scale(0.97)，150ms 弹性动画 |
| 文字 | "▶ 开始游戏", 18vp, Medium, `#FFFFFF` |

**交互逻辑**：
- 没有存档 → 直接按当前难度生成新题，跳转 GamePage
- 有存档且**同难度** → 弹窗"覆盖当前游戏？[继续游戏] [新游戏]"
- 有存档但**不同难度** → 直接生成新题（不覆盖，因为是不同难度）

### ③ 继续游戏按钮（条件显示）

**只在有存档时显示**。描边样式（区别于主按钮），信息更丰富——玩家一眼就能知道上次玩到哪了。

| 属性 | 值 |
|------|------|
| 背景 | `#FFFFFF`，描边 1.5vp `#1976D2` |
| 左侧 | "▶ 继续游戏"（主文字）+ 副行"困难 · 47%"（难度+进度） |
| 右侧 | "00:23"（已用时间） |
| 间距 | 与主按钮间距 12vp |

**长按**可弹出"放弃当前游戏？"确认框——防误触但也给出放弃的入口。

### ④ 难度选择器（SegmentButton）

5 段胶囊按钮，底部有**颜色指示条**（每个难度对应一个颜色）。

| 难度 | 色标 | 含义 |
|------|------|------|
| 🟢 简单 | `#4CAF50` 绿 | 轻松愉快 |
| 🔵 中等 | `#2196F3` 蓝 | 标准体验 |
| 🟡 困难 | `#FF9800` 橙 | 有些挑战 |
| 🟠 专家 | `#F57C00` 深橙 | 硬核玩家 |
| 🔴 地狱 | `#F44336` 红 | 自讨苦吃 |

- 默认选中**上次玩的难度**，首次进入默认"中等"
- 用 `@StorageLink('lastDifficulty')` 持久化
- 选中项白色背景 + 阴影，未选中灰色背景

### ⑤ 底部功能入口

📊 统计 + ⚙️ 设置，`Row` + `SpaceEvenly` 均匀分布。图标 22vp + 文字 12vp，颜色 `#616161`。

# 二、游戏页面（GamePage）

## 1. 整体结构

竖屏布局，从上到下 4 个区域。核心设计原则：**棋盘占最大空间，操作区紧凑但手指够得着**。

```
┌──────────────────────────────────┐
│  困难        00:23         ✕ 1/3 │  ① 顶部信息栏
├──────────────────────────────────┤
│  ┌─────────────────────────────┐ │
│  │                             │ │
│  │                             │ │
│  │        9×9 数独棋盘          │ │  ② 棋盘区（必须正方形）
│  │                             │ │
│  │                             │ │
│  │                             │ │
│  └─────────────────────────────┘ │
├──────────────────────────────────┤
│  ✏️ 笔记模式                      │  手势提示条（仅滑动时显示）
├──────────────────────────────────┤
│  ┌─────┬─────┬──────────────┐    │
│  │↩撤销│↪前进│  1   2   3   │    │
│  ├─────┼─────┤              │    │  ③ 操作区
│  │🗑擦除│💡提示│  4   5   6   │    │     左2列 = 辅助按钮
│  ├─────┼─────┤              │    │     右3列 = 手势键盘
│  │⚙设置│(预留)│  7   8   9   │    │
│  └─────┴─────┴──────────────┘    │
└──────────────────────────────────┘
```

**为什么操作区在底部？** 单手持机时，大拇指自然停留在屏幕下半部分。棋盘虽然在中上方（需要看），但操作区在底部（需要够到）。

## 2. 各区域详细设计

### ① 顶部信息栏

一行三段，`Row` + `SpaceBetween`，高 48vp。

| 位置 | 内容 | 样式 |
|------|------|------|
| 左 | 难度标签（如"困难"） | 14vp, 难度对应色（见主页 §④ 色标） |
| 中 | 计时器 `MM:SS` | 18vp, Medium, `#212121` |
| 右 | 错误计数 `✕ 1` | 14vp, 1次红色 / 2次橙色 / 3次深红 |

**计时器规则**：`onPageShow` 开始计时，`onPageHide` 暂停，页面返回继续。存档时一起保存。

**错误限制**：3 次机会。第 3 次错误后弹出"游戏失败"弹窗（可重开或返回）。错误次数用不同颜色提醒紧迫感。

### ② 数独棋盘

**必须是正方形**，这是数独 UI 的铁律。

```
尺寸计算：
  屏幕宽 360vp - 两侧边距 16vp×2 = 328vp
  单元格 ≈ 328 / 9 ≈ 36vp
```

#### 棋盘结构：宫格嵌套

```
┌─────────┬─────────┬─────────┐
│ · · · │ · · · │ · · · │
│ · · · │ · · · │ · · · │     宫格粗线 = 2vp, #212121
│ · · · │ · · · │ · · · │     单元格细线 = 0.5vp, #BDBDBD
├─────────┼─────────┼─────────┤
│ · · · │ · · · │ · · · │     每个 · 是一个 SudokuCell
│ · · · │ · · · │ · · · │
│ · · · │ · · · │ · · · │
├─────────┼─────────┼─────────┤
│ · · · │ · · · │ · · · │
│ · · · │ · · · │ · · · │
│ · · · │ · · · │ · · · │
└─────────┴─────────┴─────────┘
```

推荐实现方式：3×3 Grid 嵌套 3×3 Grid。外层 Grid 放 9 个宫格（粗线边框），每个宫格内部是 3×3 Grid 放 9 个单元格（细线边框）。这样粗/细线的渲染自然分层。

#### 单元格的 4 种视觉状态

| 状态 | 文字颜色 | 字体 | 附加效果 |
|------|---------|------|---------|
| **给定数字**（GIVEN） | `#212121` 黑 | Bold | 不可修改，点击不会进入编辑态 |
| **用户填入** | `#1976D2` 蓝 | Normal | 可修改、可擦除 |
| **错误数字**（ERROR） | `#D32F2F` 红 | Normal | 冲突高亮，吸引注意力 |
| **笔记** | `#757575` 灰 | 10vp | 3×3 排列在格内；ELIMINATED 态叠加删除线 `#BDBDBD` |

**给定数字用黑色 Bold、用户填入用蓝色 Normal**——这个颜色区分非常重要，让玩家一眼就知道哪些是题目给的、哪些是自己填的。

#### 笔记的 3×3 微型排列

一个格子内部可以同时显示多个笔记数字，按 3×3 排列：

```
┌─────────┐
│ 1  2  3 │     每个数字 10vp
│ 4  ₅  6 │     ACTIVE = 灰色正常显示
│ 7  8  9 │     ELIMINATED = 浅灰 + 删除线
└─────────┘     EMPTY = 不显示
```

#### 高亮系统

点击一个格子后，相关格子会高亮。多种高亮可能同时存在，按**优先级从高到低**渲染：

| 优先级 | 类型 | 背景色 | 触发条件 |
|--------|------|--------|---------|
| 1（最高） | 选中格 | `#BBDEFB` 浅蓝 | 当前点击的格子 |
| 2 | 同数字 | `#C8E6C9` 浅绿 | 棋盘上所有和选中格相同数字的格子 |
| 3 | 错误格 | `#FFCDD2` 浅红 | 有冲突的格子 |
| 4 | 同行列宫 | `#E3F2FD` 极浅蓝 | 选中格的同行 + 同列 + 同宫 |
| 5（最低） | 默认 | `#FFFFFF` 白 | 其他格子 |

**同数字高亮**是一个体验加分项——点击一个 7，所有 7 都变绿，帮助玩家快速扫描某个数字的分布。

### 手势模式提示条

位于棋盘和操作区之间的一条窄带，**平时隐藏（高度 0），滑动时滑出（高度 28vp）**，松手后 150ms 淡出。

这个提示条告诉玩家当前手势是什么模式：

| 检测到的手势方向 | 提示内容 | 背景色 |
|----------------|---------|--------|
| 上滑（笔记） | `✏️ 笔记模式` | `#E3F2FD` 浅蓝 |
| 右滑（删除线） | `～～ 删除线模式` | `#FFF3E0` 浅橙 |
| 左滑（擦除） | `🧹 擦除笔记` | `#F5F5F5` 浅灰 |
| 下滑（预留） | `⬇️ 预留功能` | `#F5F5F5` 浅灰 |

**设计理由**：手势操作的最大问题是"我刚才滑的方向对不对？"。这个提示条提供即时反馈，让玩家建立肌肉记忆。等熟练了自然就不看了。

### ③ 操作区

5 列 × 3 行的网格，左 2 列是辅助功能按钮，右 3 列是手势数字键盘。

#### 辅助按钮（2×3 网格）

| 行 | 列 0 | 列 1 |
|----|------|------|
| 0 | ↩️ **撤销** | ↪️ **前进** |
| 1 | 🗑️ **擦除** | 💡 **提示** |
| 2 | ⚙️ **设置** | — 预留 |

| 按钮 | 点击效果 | 灰态条件 |
|------|---------|---------|
| ↩️ 撤销 | 回退上一步操作 | 操作栈为空 |
| ↪️ 前进 | 恢复撤销的操作 | 前进栈为空 |
| 🗑️ 擦除 | 清空选中格（value=0, notes=0） | 选中格是 GIVEN / 无内容 |
| 💡 提示 | 填入正确数字 + 标记 LOCKED | 选中格已有正确数字 / 是 GIVEN |
| ⚙️ 设置 | 打开设置面板 | — |

按钮样式：60vp 宽，图标 20vp + 文字 10vp，圆角 8vp，按压态 scale(0.95) + 背景加深。

## 3. 3×3 手势键盘——核心创新

### 设计理念

传统数独 App 要么用 9 个按钮（占空间），要么用模式切换（"笔记模式"开关）。我们的方案：**每个键支持点击 + 四方向滑动，一根手指完成所有操作**。

不需要模式切换按钮——手势就是模式。

### 手势映射

```
                ▲ 上滑 = 切换笔记
                │      （EMPTY ↔ ACTIVE）
                │
 左滑 ◄──────── ● ────────► 右滑 = 切换删除线
 擦除笔记        │    点击         （EMPTY/ACTIVE → ELIMINATED
 (→EMPTY)       │   = 填数         ELIMINATED → EMPTY）
                │
                ▼ 下滑 = 预留（未来功能）
```

一张表说清楚：

| 手势 | 操作名 | 效果 | 适用场景 |
|------|--------|------|---------|
| **点击** | 填数 | 选中格填入该数字；已有相同数字则擦除（toggle） | 最常用 |
| **上滑** | 笔记 | 切换笔记：`EMPTY ↔ ACTIVE`；如果格子有 value，先清 value | 标记候选数 |
| **右滑** | 删除线 | 切换删除线：`非 ELIM → ELIMINATED`，`ELIMINATED → EMPTY` | 排除候选 |
| **左滑** | 擦除 | 该数字的笔记状态 → `EMPTY` | 清除标记 |
| **下滑** | 预留 | 暂无功能，留给未来扩展 | — |

### 手势判定算法

用触摸的起点和终点计算方向，核心是 `atan2` + 角度划分：

```typescript
enum GestureAction { TAP, SWIPE_UP, SWIPE_RIGHT, SWIPE_DOWN, SWIPE_LEFT }

function resolveGesture(dx: number, dy: number, threshold: number = 30): GestureAction {
  // 距离太短 → 视为点击
  if (Math.sqrt(dx * dx + dy * dy) < threshold) {
    return GestureAction.TAP
  }

  // 用 atan2 算角度（注意 y 轴翻转：屏幕向下是正）
  const angle = Math.atan2(-dy, dx) * 180 / Math.PI
  // angle 范围：-180° ~ 180°
  //   右 = 0°    上 = 90°    左 = ±180°    下 = -90°

  if (angle > 45 && angle <= 135)   return GestureAction.SWIPE_UP
  if (angle > -45 && angle <= 45)   return GestureAction.SWIPE_RIGHT
  if (angle > -135 && angle <= -45) return GestureAction.SWIPE_DOWN
  return GestureAction.SWIPE_LEFT  // angle > 135 || angle <= -135
}
```

**threshold = 30vp** 是经验值——太小容易误判点击为滑动，太大滑动手感迟钝。可以后续调优。

**为什么用 `atan2` 不用简单的 dx/dy 比较？** 因为 atan2 给出连续的角度值，4 个方向的判定边界是均匀的 90° 扇形。用 dx/dy 比较需要很多 if-else，而且对角线方向（45°）的判定不自然。

### 键的视觉设计

```
┌─────────┐
│         │
│    7    │   ← 数字 28vp, Medium, #212121
│         │
│    2    │   ← 剩余计数 11vp, #9E9E9E（"还剩几个7没填"）
└─────────┘
    12vp 圆角
```

| 状态 | 样式 |
|------|------|
| 正常 | 白底 `#FFFFFF` |
| 按下 | scale(0.92) + 背景加深 |
| 数字已满（9个都填了） | opacity 0.5 + 背景 `#E0E0E0` |

**剩余计数**是一个很好的辅助信息——当某个数字已经填满 9 个了，键变灰，玩家不需要自己去数。

### 视觉反馈——滑动时的方向提示

手指在键上滑动时，**键的背景色**随方向变化，提供即时视觉反馈：

| 方向 | 键背景色 |
|------|---------|
| 还没超过 threshold | 正常（白） |
| 上滑（笔记） | `#E3F2FD` 浅蓝 |
| 右滑（删除线） | `#FFF3E0` 浅橙 |
| 左滑（擦除） | `#F5F5F5` 浅灰 |
| 下滑（预留） | `#F5F5F5` 浅灰 |

同时，棋盘和操作区之间的**手势提示条**也会滑出，显示文字提示（见前面 §手势模式提示条）。

### 首次使用引导

第一次进入游戏页时，弹出半透明遮罩，聚焦数字键区域：

```
┌─────────────────────────────┐

│                              │
│   ┌───────────────────────┐  │
│   │  ● 点击 → 填入数字    │  │   ← 聚光灯效果
│   │  ↑ 上滑 → 标记笔记    │  │      只有键盘区域是亮的
│   │  → 右滑 → 标记删除线  │  │
│   │  ← 左滑 → 擦除标记    │  │
│   │                        │  │
│   │     [ 知道了 ]         │  │
```

## 4. 交互流程详解

以下是每种手势操作的**完整流程**，包括边界条件处理。

### 4.1 填数（点击键盘数字）

```
用户点击数字键 d（例如 7）
  │
  ├── 没有选中格 → 无操作
  ├── 选中格是 GIVEN → 无操作（给定数字不可修改）
  ├── 选中格的 value === d → 擦除（value = 0，toggle 行为）
  └── 其他情况：
      ├── value = d
      ├── 清空该格所有 notes（填了数字就不需要笔记了）
      ├── 检查冲突 → 有冲突则标记 ERROR，错误计数 +1
      └── 检查是否完成 → 81 格全填满且无冲突 → 🎉 胜利弹窗
```

**toggle 行为**（点击已填的同一个数字就擦除）非常实用——不需要先切到擦除再操作，直接"再点一次"就撤销了。

### 4.2 笔记（上滑）

```
用户上滑数字键 d
  │
  ├── 没有选中格 → 无操作
  ├── 选中格是 GIVEN → 无操作
  ├── 选中格有 value → 先清空 value（转为笔记模式）
  └── 切换笔记状态：
      ├── EMPTY → ACTIVE        （标记这个候选数）
      ├── ACTIVE → EMPTY        （取消标记）
      └── ELIMINATED → ACTIVE   （从"排除"恢复为"激活"）
```

### 4.3 删除线（右滑）

```
用户右滑数字键 d
  │
  ├── 没有选中格 / 是 GIVEN → 无操作
  └── 切换删除线：
      ├── EMPTY → ELIMINATED       （直接标记排除）
      ├── ACTIVE → ELIMINATED      （从激活变为排除）
      └── ELIMINATED → EMPTY       （取消排除标记）
```

**删除线和笔记的关系**：笔记的 ACTIVE 表示"我觉得这个数可能"，ELIMINATED 表示"我确定这个数不行"。这个三态设计来自第 01 文档的 NoteState 位图（每个数字 2 bit）。

### 4.4 擦除笔记（左滑）

```
用户左滑数字键 d
  │
  ├── 没有选中格 / 是 GIVEN → 无操作
  └── 该数字的 notes 状态 → EMPTY（无论之前是什么状态）
```

最简单的操作——"我不想要这个标记了"。

### 4.5 工具栏擦除（🗑️ 按钮）

与手势的"左滑擦除"不同，工具栏擦除是**清空整个格子**：

```
用户点击 🗑️ 擦除按钮
  │
  ├── 选中格是 GIVEN → 无操作
  └── 清空选中格：value = 0, notes = 0（全部归零）
```

### 4.6 选中格切换

```
用户点击棋盘上的某个格子 C
  │
  ├── C 就是当前选中格 → 取消选中（selectedCell = null）
  └── C 不是当前选中格 → 选中 C，更新高亮
      └── 无论 C 是否 GIVEN，都可以被选中和高亮
         （GIVEN 格可以选中查看高亮，但不能修改）
```

## 6. 左右手自适应

通过 `@kit.MultimodalAwarenessKit` 检测操作手（详见 [06-AI与分布式能力调研](./06-AI与分布式能力调研.md#4)），自动**镜像翻转**操作区布局。

| 适配项 | 右手模式（默认） | 左手模式 |
|--------|------------------|----------|
| 手势键盘位置 | 偏右（右 3 列） | 偏左（左 3 列） |
| 辅助按钮列 | 在键盘左边 | 在键盘右边 |
| AI 教练按钮 🤖 | 右上角 | 左上角 |

**核心思路**：惯用手一侧放高频操作（键盘），另一侧放低频操作（辅助按钮）。

不支持手势检测的设备（API 版本不够，或返回错误码 `801`）**静默 fallback 到右手模式**——不弹错误提示，用户无感。
