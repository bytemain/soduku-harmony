## 1. 什么是鸿蒙

### 1.1 简要历史

| 时间 | 事件 |
|------|------|
| 2019 | 华为发布 HarmonyOS 1.0 |
| 2021 | HarmonyOS 2.0，首次用于手机 |
| 2023 | HarmonyOS 4.0 |
| 2024 | **HarmonyOS NEXT**（纯血鸿蒙），不再兼容 Android |

### 1.2 HarmonyOS NEXT 特点

- **纯自研系统**：不再兼容 Android APK
- **ArkTS 语言**：基于 TypeScript 扩展的开发语言
- **ArkUI 框架**：声明式 UI 框架（类似 Flutter / SwiftUI / Jetpack Compose）
- **分布式能力**：支持多设备协同
- **统一开发**：一套代码适配手机、平板、手表、电视等

### 1.3 与其他平台的对比

| 特性 | 鸿蒙 | Android | iOS |
|------|------|---------|-----|
| 开发语言 | ArkTS | Kotlin/Java | Swift |
| UI 框架 | ArkUI（声明式） | Jetpack Compose | SwiftUI |
| 开发工具 | DevEco Studio | Android Studio | Xcode |
| 应用格式 | .hap | .apk | .ipa |
| UI 范式 | 声明式 | 声明式/命令式 | 声明式/命令式 |

## 3. ArkTS 语言基础

### 3.1 ArkTS 是什么？

ArkTS = **TypeScript + 声明式 UI 扩展 + 一些限制**

如果你熟悉 TypeScript，ArkTS 上手非常快。主要区别：

| 特性 | TypeScript | ArkTS |
|------|-----------|-------|
| 类型系统 | 动态类型推断 | 更严格的静态类型 |
| UI | 无 | 内置声明式 UI 语法 |
| 装饰器 | 支持 | 大量使用（@Entry, @Component, @State...） |
| any 类型 | 允许 | **不允许**（更严格） |
| 动态属性 | 允许 | **不允许** |

### 3.2 基本语法

```typescript
// 变量声明
let name: string = "Sudoku"
let count: number = 9
let isPlaying: boolean = true
let board: number[][] = []

// 常量
const MAX_SIZE: number = 9

// 函数
function add(a: number, b: number): number {
  return a + b
}

// 箭头函数
const multiply = (a: number, b: number): number => a * b

// 接口
interface Cell {
  row: number
  col: number
  value: number
}

// 类
class SudokuGame {
  private board: number[][] = []
  private difficulty: number = 1

  constructor(difficulty: number) {
    this.difficulty = difficulty
  }

  getBoard(): number[][] {
    return this.board
  }
}

// 枚举
enum Difficulty {
  EASY = 1,
  MEDIUM = 2,
  HARD = 3
}

// 数组操作
let numbers: number[] = [1, 2, 3, 4, 5]
numbers.push(6)
numbers.filter(n => n > 3)  // [4, 5, 6]
numbers.map(n => n * 2)     // [2, 4, 6, 8, 10, 12]

// 循环
for (let i = 0; i < 9; i++) { /* ... */ }
for (let item of numbers) { /* ... */ }
numbers.forEach((item, index) => { /* ... */ })
```

### 3.3 ArkTS 的限制（与标准 TypeScript 不同）

```typescript
// ❌ 不允许 any 类型
let x: any = 123  // 错误！

// ❌ 不允许动态属性访问
let obj = { name: "test" }
let key = "name"
obj[key]  // 错误！

// ❌ 不允许 eval
eval("code")  // 错误！

// ✅ 替代方案：使用 Map
let map = new Map<string, string>()
map.set("name", "test")
map.get("name")
```

## 4. ArkUI 框架核心概念

### 4.1 声明式 UI 是什么？

**命令式 UI（传统方式）：**
```
1. 创建一个按钮对象
2. 设置按钮的文字为 "点击"
3. 设置按钮的颜色为蓝色
4. 把按钮添加到页面上
5. 当数据变化时，手动更新按钮的文字
```

**声明式 UI（ArkUI 的方式）：**
```
描述："有一个蓝色按钮，显示 message 变量的内容"
当 message 变化时 → 框架自动更新 UI
```

### 4.2 第一个组件

让我们解读你项目中已有的代码：

```typescript
@Entry        // 装饰器：标记这是一个页面入口组件
@Component    // 装饰器：标记这是一个自定义组件
struct Index { // struct：组件使用 struct 而不是 class

  @State message: string = 'Hello World'  // @State：状态变量，变化时自动刷新 UI

  build() {   // build()：描述 UI 的方法，必须有
    RelativeContainer() {   // 布局容器
      Text(this.message)    // 文本组件，显示 message 的值
        .fontSize(50)       // 链式调用设置属性
        .fontWeight(FontWeight.Bold)
        .onClick(() => {           // 点击事件
          this.message = 'Welcome' // 修改 @State 变量 → 自动刷新 UI
        })
    }
    .height('100%')
    .width('100%')
  }
}
```

### 4.3 核心装饰器速查表

| 装饰器 | 用途 | 说明 |
|--------|------|------|
| `@Entry` | 页面入口 | 每个页面的根组件必须标记 |
| `@Component` | 自定义组件 | 所有自定义组件必须标记 |
| `@State` | 组件内部状态 | 变化时触发 UI 重新渲染 |
| `@Prop` | 父→子单向传递 | 父组件传给子组件的数据，子组件不能修改源头 |
| `@Link` | 父↔子双向绑定 | 子组件修改会同步到父组件 |
| `@Provide` / `@Consume` | 跨组件共享 | 类似 React Context |
| `@Observed` / `@ObjectLink` | 嵌套对象观察 | 监听对象内部属性变化 |
| `@Builder` | 可复用 UI 片段 | 类似函数组件 |
| `@Styles` | 可复用样式 | 抽取公共样式 |
| `@Extend` | 扩展组件样式 | 为内置组件添加自定义样式方法 |

### 4.4 组件的基本结构

```typescript
@Component
struct MyComponent {
  // 1. 状态声明
  @State count: number = 0
  @Prop title: string = ''

  // 2. 生命周期
  aboutToAppear(): void {
    // 组件即将出现时调用（类似 onMounted / componentDidMount）
    console.log('组件即将显示')
  }

  aboutToDisappear(): void {
    // 组件即将销毁时调用
    console.log('组件即将销毁')
  }

  // 3. 自定义方法
  increment(): void {
    this.count++
  }

  // 4. UI 构建（必须）
  build() {
    Column() {
      Text(this.title)
        .fontSize(24)

      Text(`计数: ${this.count}`)
        .fontSize(20)

      Button('增加')
        .onClick(() => this.increment())
    }
  }
}
```

## 6. 组件生命周期

### 6.1 页面生命周期（@Entry 组件）

```
┌────────────────────────────────────────────┐
│                 创建阶段                     │
│                                              │
│  aboutToAppear()  ← 在 build() 之前调用      │
│      ↓              初始化数据的好时机         │
│  build()          ← 首次构建 UI              │
│      ↓                                       │
│  onPageShow()     ← 页面显示时               │
│                                              │
├────────────────────────────────────────────┤
│                 运行阶段                     │
│                                              │
│  [用户交互]                                  │
│      ↓                                       │
│  @State 变化 → build() 重新执行              │
│                                              │
│  onPageHide()     ← 页面隐藏时（如跳转）     │
│  onPageShow()     ← 页面再次显示             │
│                                              │
├────────────────────────────────────────────┤
│                 销毁阶段                     │
│                                              │
│  aboutToDisappear() ← 组件即将销毁           │
│                       释放资源的好时机        │
└────────────────────────────────────────────┘
```

### 6.2 实际示例

```typescript
@Entry
@Component
struct GamePage {
  @State timer: number = 0
  private intervalId: number = -1

  // 组件即将出现
  aboutToAppear(): void {
    console.log('游戏页面即将出现')
    this.loadGame()
  }

  // 页面显示
  onPageShow(): void {
    console.log('页面显示，开始计时')
    this.intervalId = setInterval(() => {
      this.timer++
    }, 1000)
  }

  // 页面隐藏
  onPageHide(): void {
    console.log('页面隐藏，暂停计时')
    clearInterval(this.intervalId)
  }

  // 组件即将销毁
  aboutToDisappear(): void {
    console.log('清理资源')
    clearInterval(this.intervalId)
  }

  loadGame(): void {
    // 初始化游戏...
  }

  build() {
    Column() {
      Text(`用时: ${this.timer}秒`)
    }
  }
}
```

## 7. 状态管理

### 7.1 @State — 组件内部状态

> 当 @State 变量的值变化时，框架会自动重新执行 build() 来更新 UI。

```typescript
@Component
struct Counter {
  @State count: number = 0  // 声明状态变量

  build() {
    Column() {
      Text(`${this.count}`)     // 读取状态
      Button('+1')
        .onClick(() => {
          this.count++          // 修改状态 → 自动刷新 UI
        })
    }
  }
}
```

**注意事项：**
```typescript
// ✅ 这些变化能被检测到：
this.count = 10                    // 基本类型赋值
this.list = [1, 2, 3]             // 数组整体替换
this.list.push(4)                  // 数组方法（push/pop/splice 等）

// ❌ 这些变化 *不能* 被检测到：
this.list[0] = 99                 // 直接修改数组元素（需要用 splice）
this.obj.property = 'new'         // 嵌套对象属性变化（需要 @Observed）
```

### 7.2 @Prop — 父到子单向传递

```typescript
// 父组件
@Component
struct Parent {
  @State score: number = 0

  build() {
    Column() {
      Child({ playerScore: this.score })  // 传递给子组件
      Button('加分').onClick(() => this.score++)
    }
  }
}

// 子组件
@Component
struct Child {
  @Prop playerScore: number = 0  // 接收父组件数据（单向）

  build() {
    Text(`得分: ${this.playerScore}`)
  }
}
```

### 7.3 @Link — 父子双向绑定

```typescript
// 父组件
@Component
struct Parent {
  @State selectedNumber: number = 0

  build() {
    Column() {
      Text(`选中: ${this.selectedNumber}`)
      // ⚠️ 传给 @Link 变量时，必须用 $ 前缀：$selectedNumber
      // this.selectedNumber → 传的是"值"（单向，适合 @Prop）
      // this.$selectedNumber → 传的是"引用"（双向，适合 @Link）
      NumberPicker({ selected: $selectedNumber })
    }
  }
}

// 子组件
@Component
struct NumberPicker {
  @Link selected: number  // 双向绑定，修改会同步到父组件

  build() {
    Row() {
      ForEach([1,2,3,4,5,6,7,8,9], (num: number) => {
        Button(`${num}`)
          .onClick(() => {
            this.selected = num  // 修改会同步到父组件的 selectedNumber
          })
      })
    }
  }
}
```

### 7.4 @Observed 和 @ObjectLink — 嵌套对象

```typescript
// 需要被观察的类
@Observed
class CellData {
  value: number = 0
  isGiven: boolean = false
  isError: boolean = false

  constructor(value: number, isGiven: boolean) {
    this.value = value
    this.isGiven = isGiven
  }
}

// 父组件
@Component
struct Board {
  @State cells: CellData[] = []

  build() {
    ForEach(this.cells, (cell: CellData, index: number) => {
      CellView({ cell: cell })  // 传递对象
    })
  }
}

// 子组件
@Component
struct CellView {
  @ObjectLink cell: CellData  // 观察对象内部属性变化

  build() {
    Text(`${this.cell.value}`)
      .fontColor(this.cell.isError ? Color.Red : Color.Black)
  }
}
```

### 7.5 @Provide / @Consume — 跨层级共享

```typescript
// 祖先组件
@Component
struct GamePage {
  @Provide('gameState') state: string = 'playing'

  build() {
    Column() {
      Board()  // Board 内部的子组件也能访问 gameState
    }
  }
}

// 任意后代组件（无需逐层传递）
@Component
struct Cell {
  @Consume('gameState') state: string  // 直接获取祖先的数据

  build() {
    Text('...')
      .enabled(this.state === 'playing')
  }
}
```

### 7.6 状态管理选择指南

```
问题：这个数据在哪里用？

只在当前组件用
  → 使用 @State

需要传给子组件，子组件只读
  → 使用 @Prop

需要传给子组件，子组件也能修改
  → 使用 @Link

需要跨越多层组件共享
  → 使用 @Provide / @Consume

需要监听对象内部属性变化
  → 使用 @Observed + @ObjectLink
```

### 10.3 数据持久化（保存游戏进度）

```typescript
import { preferences } from '@kit.ArkData'

// 保存数据
async function saveGame(context: Context, boardData: string): Promise<void> {
  let store = preferences.getPreferencesSync(context, { name: 'sudoku_save' })
  store.putSync('board', boardData)
  store.putSync('timer', 120)
  await store.flush()
}

// 读取数据
function loadGame(context: Context): string {
  let store = preferences.getPreferencesSync(context, { name: 'sudoku_save' })
  return store.getSync('board', '') as string
}
```

### 10.5 自定义弹窗

```typescript
@CustomDialog
struct GameCompleteDialog {
  controller: CustomDialogController
  @Prop time: number = 0

  build() {
    Column({ space: 20 }) {
      Text('🎉 恭喜完成！')
        .fontSize(24)
        .fontWeight(FontWeight.Bold)

      Text(`用时: ${Math.floor(this.time / 60)}分${this.time % 60}秒`)
        .fontSize(18)

      Button('再来一局')
        .onClick(() => {
          this.controller.close()
        })
    }
    .padding(30)
  }

@Component
struct GamePage {
  dialogController: CustomDialogController = new CustomDialogController({
    builder: GameCompleteDialog({ time: this.timer }),
    autoCancel: false
  })

  showComplete(): void {
    this.dialogController.open()
  }
}
```
