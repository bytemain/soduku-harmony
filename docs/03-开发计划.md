# 📋 数独 App 开发计划

> 基于 HarmonyOS NEXT + ArkTS 的数独游戏 App 开发计划

---

## 项目信息

| 项目 | 说明 |
|------|------|
| 项目名称 | Soduku（数独） |
| 包名 | com.bytemain.soduku |
| 开发框架 | HarmonyOS NEXT, ArkTS, ArkUI |
| 目标设备 | 手机、平板 |

---

## 功能清单

### Phase 1 — MVP（最小可用版本）

- [ ] **数独棋盘渲染**：9×9 网格，区分宫格边框
- [ ] **数独生成算法**：能生成有唯一解的数独题目
- [ ] **数独求解算法**：回溯法求解
- [ ] **难度选择**：简单 / 中等 / 困难
- [ ] **数字输入**：点击格子 → 选择数字填入
- [ ] **冲突检测**：实时标红冲突的数字
- [ ] **完成判定**：填满且无冲突时弹出胜利提示

### Phase 2 — 体验增强

- [ ] **笔记模式**：在格子中标记候选数字
- [ ] **撤销/重做**：支持操作回退
- [ ] **计时器**：记录用时
- [ ] **高亮系统**：选中格子 → 高亮同行/列/宫/同数字
- [ ] **错误计数**：限制错误次数
- [ ] **新游戏**：不退出页面重新开始

### Phase 3 — 完善功能

- [ ] **提示系统**：给出下一步提示（基于解题技巧）
- [ ] **保存/继续**：游戏进度持久化
- [ ] **统计数据**：记录游戏历史、最佳时间
- [ ] **动画效果**：填入/错误/完成动画
- [ ] **深色模式**：适配系统深色主题
- [ ] **音效/触感**：填数音效、错误震动

### Phase 4 — 高级功能

- [ ] **每日挑战**：每天一道题
- [ ] **成就系统**：解锁成就
- [ ] **多种数独变体**：对角线数独、杀手数独等

## 建议的文件结构

```
entry/src/main/ets/
├── entryability/
│   └── EntryAbility.ets         # 应用入口
├── pages/
│   ├── Index.ets                # 主页（难度选择）
│   └── GamePage.ets             # 游戏页面
├── components/
│   ├── SudokuBoard.ets          # 数独棋盘组件
│   ├── SudokuCell.ets           # 单个格子组件
│   ├── NumberPad.ets            # 数字输入键盘
│   ├── GameToolbar.ets          # 工具栏（撤销/笔记/提示等）
│   └── TimerDisplay.ets         # 计时器显示
├── models/
│   ├── CellData.ets             # 格子数据模型
│   ├── GameState.ets            # 游戏状态模型
│   └── GameHistory.ets          # 操作历史（用于撤销）
├── utils/
│   ├── SudokuGenerator.ets      # 数独生成器
│   ├── SudokuSolver.ets         # 数独求解器
│   ├── SudokuValidator.ets      # 验证工具
│   └── DifficultyEvaluator.ets  # 难度评估
├── constants/
│   └── GameConstants.ets        # 常量定义（颜色、尺寸等）
└── services/
    └── StorageService.ets       # 数据持久化服务
```

---

## 开发顺序建议

```
第 1 步：实现数独核心算法（生成 + 求解 + 验证）
    ↓ 可以用单元测试验证正确性
第 2 步：创建数据模型（CellData, GameState）
    ↓
第 3 步：实现棋盘 UI（Grid 布局 + 格子组件）
    ↓ 先用写死的数据测试 UI
第 4 步：实现数字输入键盘
    ↓
第 5 步：连接算法和 UI（选择难度 → 生成题目 → 显示）
    ↓
第 6 步：实现冲突检测和完成判定
    ↓ 到此 MVP 完成 🎉
第 7 步：逐步添加 Phase 2/3 的功能
```
